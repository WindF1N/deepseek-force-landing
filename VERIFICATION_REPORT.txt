PRIMARY FIX: Tool Results Deduplication
═══════════════════════════════════════════════════════════════════════════

PROBLEM:
  In persistent chat mode, orchestrator was collecting ALL tool results from
  message history and resending them with every request. DeepSeek had already
  seen these results (stored via parent_message_id), causing:
  
  • Prompt bloat: 100KB+ payloads instead of <100 chars
  • Context pollution: Redundant data overwhelming the model
  • Token waste: Paying to resend old tool outputs every turn

SOLUTION (orchestrator/api.py lines 200-220):
  Changed logic to collect ONLY NEW tool results - those that appear AFTER
  the last assistant message in history. These are fresh results that
  DeepSeek hasn't seen yet via parent_message_id.

CODE CHANGE:
  ```python
  # Find last assistant message index
  last_assistant_idx = None
  for i, m in enumerate(reversed(messages)):
      if m.get("role") == "assistant":
          last_assistant_idx = len(messages) - 1 - i
          break
  
  # Collect only NEW tool results (after last assistant)
  for i, m in enumerate(messages):
      if m.get("role") == "user" and not last_user_msg:
          last_user_msg = m
      elif m.get("role") == "tool":
          if last_assistant_idx is None or i > last_assistant_idx:
              tool_results.append(m)
  ```
