Технологии

Что под капотом

Не магия — инженерные решения, проверенные на тысячах запросов.

🗳️

Self-Consistency Voting

Каждый запрос уходит в 3+ параллельных окна DeepSeek. Ответы сравниваются — консенсусный вариант побеждает. Одиночные ошибки отсеиваются автоматически.

# 3 окна → 3 ответа → 1 консенсус
raws = await bridge.fan_out(prompt, n=3)
decision = validator.vote(raws, tools)
# → kind: "tool_calls", votes: 3/3 ✓
🛡️

Anti-Hallucination Gates

Три уровня защиты: детектор ложных «файл создан» без вызова инструмента, stall guard для забытых tool calls, и completion gate — проверка реального завершения задачи.

# Модель соврала «готово» без write_file?
if claims_done(response) and not write_ever_called:
  retry # → принудительный tool call
🔧

Truncated Salvage

DeepSeek обрезает огромные write_file на полуслове. Наш парсер восстанавливает имя инструмента, собранные аргументы и даже оборванный ключ из битого JSON.

# JSON оборван на полуслове
salvaged = salvage_truncated(broken_json)
# → {name: "write_file", arguments: {path: "x", content: "…"}}
♻️

Chat Rotation + Суммаризация

Контекст DeepSeek ограничен ~100 сообщениями. Мы автоматически создаём новый чат, суммаризируем предыдущий через helper-окно, и продолжаем диалог бесконечно.

🔀

Multi-Chat Architecture

DeepSeek не позволяет сменить model_type внутри чата. Мы держим три параллельных чата: Primary (агент), Vision (картинки), Search (поиск). Результаты helper-чатов возвращаются в основной.

Chrome-расширение автоматически решает Proof-of-Work на chat.deepseek.com. Никаких API-ключей, никаких биллингов — чистый бесплатный доступ к DeepSeek Instant, Expert и Vision.

Проблемы

С какими проблемами сталкиваются AI-агенты

Прямой доступ к DeepSeek — это мощно, но без оркестрации вы рискуете получить эти проблемы.

🎭

Галлюцинации

Модель уверенно заявляет «файл создан», хотя write_file не вызывался. Одиночное окно не может проверить само себя.

✂️

Обрезанные ответы

DeepSeek обрывает JSON на полуслове при больших объёмах. Без salvage-парсера вы теряете результат и вынуждены повторять запрос.

🧱

Лимит контекста

~100 сообщений — и чат заканчивается. Без ротации и суммаризации ваш диалог обрывается на самом важном месте.

Как DeepSeek Force это решает

Три ключевых механизма, которые превращают уязвимый одиночный вызов в надёжную систему.

FAQ

Частые вопросы

Всё, что вы хотели знать о DeepSeek Force.

QЭто действительно бесплатно?▾

Да. DeepSeek Force использует бесплатный веб-доступ к chat.deepseek.com через Chrome-расширение. Никаких API-ключей, никаких биллингов. Единственное требование — запущенный Chrome с нашим расширением.

QЧем это лучше прямого API DeepSeek?▾

Прямой API — это один вызов, один ответ. Нет проверки, нет голосования, нет защиты от галлюцинаций. DeepSeek Force добавляет слой оркестрации: 3+ параллельных вызова, консенсус, валидация. Плюс — это бесплатно, в отличие от API за $0.14/1M токенов.

QКакой model_type используется?▾

Основной чат использует DeepSeek Instant (быстрый, для кода и агентских задач). Для картинок — Vision-окно, для поиска — Search-окно. Переключение происходит автоматически, прозрачно для пользователя.

QКак установить и запустить?▾

1. Установите Chrome-расширение из архива. 2. Запустите DeepSeek Bridge (порт 8001). 3. Запустите Orchestrator (порт 8002). 4. Укажите эндпоинт в конфиге Hermes Agent. Полная инструкция — в README репозитория.

QСколько Chrome-окон нужно?▾

Минимум 3 для Self-Consistency Voting. Bridge управляет пулом автоматически — создаёт, перезапускает, ротирует при ошибках. Вы просто указываете количество в конфиге.
