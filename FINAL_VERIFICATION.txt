CORE FIX: Tool Results Deduplication
═══════════════════════════════════════════════════════════════════════════

Location: orchestrator/api.py lines 208-223

Logic:
  1. Find last assistant message index in history
  2. Collect ONLY tool results that appear AFTER that index
  3. These are NEW results DeepSeek hasn't seen yet via parent_message_id
  
Code:
  ```python
  last_assistant_idx = None
  for i, m in enumerate(reversed(messages)):
      if m.get("role") == "assistant":
          last_assistant_idx = len(messages) - 1 - i
          break
  
  for i, m in enumerate(messages):
      if m.get("role") == "user" and not last_user_msg:
          if i > (last_assistant_idx or -1):
              last_user_msg = m
      elif m.get("role") == "tool":
          if last_assistant_idx is None or i > last_assistant_idx:
              tool_results.append(m)
  ```
